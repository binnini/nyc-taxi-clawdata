# Troubleshoot: 2026-03-21

---

## 1. Openclaw Heartbeat 미설정으로 Anthropic 크레딧 전액 소진

**증상**
- 2026-03-19 EC2 서버 기동 후 이틀 만에 Anthropic API 크레딧 전액 소진
- 2026-03-20 14:01 UTC부터 모든 API 호출이 `credit balance too low` 오류
- wikistreams reporter (별도 서버)도 2026-03-21 00:00 UTC에 동일 오류

**원인**

openclaw gateway가 기동되면 **HEARTBEAT.md 유무와 무관하게** 30분마다 자동으로 에이전트에게 heartbeat 메시지를 보낸다. 이때 openclaw.json에 설정된 모델(`claude-opus-4-6`)과 thinking 설정(`medium`)이 그대로 적용된다.

| 항목 | 값 |
|------|----|
| 모델 | `claude-opus-4-6` (가장 비싼 모델) |
| Thinking | `medium` (extended thinking, 출력 단가 적용) |
| Heartbeat 간격 | 30분 (`intervalMs: 1800000`) |
| 캐시 TTL | 5분 → 30분 간격 heartbeat은 매 호출이 **캐시 미적중** |
| 시스템 프롬프트 | 11,738 토큰 (매 호출마다 풀 가격 부과) |

소진 타임라인:
```
2026-03-19 07:01 UTC  openclaw-gateway 서비스 시작, heartbeat 시작
2026-03-19 07:31 ~    30분마다 claude-opus-4-6 API 호출 (성공)
2026-03-20 14:01 UTC  크레딧 소진 — 첫 billing 오류 기록
2026-03-21 00:00 UTC  wikistreams reporter도 동일 오류
```
→ 기동 후 약 31시간 만에 전액 소진 (heartbeat ~62회)

**조치**

1. `openclaw system heartbeat disable` 로 즉시 비활성화
2. 재시작 시에도 유지되도록 systemd service에 `ExecStartPost` 추가:

```ini
# /etc/systemd/system/openclaw-gateway.service
ExecStartPost=/bin/bash -c 'sleep 5 && /usr/bin/openclaw system heartbeat disable --token f056a5ca0e208bd14544219ad8d229fd'
```

**향후 heartbeat 사용 시 주의사항**

heartbeat을 다시 활성화할 경우:
- HEARTBEAT.md에 실제로 할 작업이 있을 때만 사용
- 모델을 `claude-haiku-4-5-20251001`로 변경 (약 50배 저렴)
- thinking 비활성화 (`thinkingLevel: none`)
- interval을 충분히 길게 설정 (예: 6시간 이상)
- 또는 heartbeat 대신 openclaw cron으로 필요 시에만 실행

```bash
# 현재 heartbeat 상태 확인
openclaw system heartbeat last --json

# 재활성화 방법 (필요 시)
openclaw system heartbeat enable
```
