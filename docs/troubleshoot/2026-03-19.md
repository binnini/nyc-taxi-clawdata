# Troubleshoot: 2026-03-19

## 1. dbt 미설치

**증상**
```
/bin/bash: line 1: dbt: command not found
```

**원인**
EC2에 dbt-duckdb가 설치되지 않은 상태.

**해결**
```bash
pip3 install dbt-duckdb --break-system-packages
export PATH=$PATH:/home/ubuntu/.local/bin
```

## 2. `read_parquet()` 다중 경로 문법 오류

**증상**
```
Binder Error: No function matches the given name and argument types 'read_parquet(VARCHAR, VARCHAR)'
```

**원인**
```sql
-- 잘못된 문법 (두 번째 인자를 별도 VARCHAR로 전달)
FROM read_parquet(
    's3://nyc-tlc/trip data/yellow_tripdata_2019-*.parquet',
    's3://nyc-tlc/trip data/yellow_tripdata_2020-*.parquet'
)
```
DuckDB `read_parquet()`은 여러 경로를 배열 `[...]`로 받아야 함.

**해결**
```sql
FROM read_parquet([
    'https://...2019-01.parquet',
    'https://...2019-02.parquet',
    ...
], union_by_name=true)
```

## 3. S3 RODA `nyc-tlc` 버킷 접근 차단

**증상**
```
HTTP Error: HTTP GET error reading 's3://nyc-tlc/...' (HTTP 403 Forbidden)
AccessDenied: Access Denied
```

**원인**
`s3://nyc-tlc/` 버킷이 익명 및 인증 요청 모두 차단하도록 버킷 정책 변경됨.
(과거 AWS RODA 공개 버킷이었으나 현재 외부 접근 불가)

**확인한 방법들과 결과**

| 방법 | 결과 |
|------|------|
| EC2(Seoul) → S3 익명(`--no-sign-request`) | AccessDenied |
| EC2(Seoul) → S3 인증 | AccessDenied |
| CloudShell(us-east-1) → S3 익명 | AccessDenied |
| CloudShell(us-east-1) → S3 인증 | AccessDenied |
| CloudShell(us-east-1) → S3 requester-pays | AccessDenied |
| `aws s3api get-bucket-policy --bucket nyc-tlc` | AccessDenied |

**해결 방향**
CloudFront URL(`https://d37ci6vzurychx.cloudfront.net/trip-data/`) 사용으로 전환.

## 4. CloudFront Geo-block (EC2 Seoul IP 차단)

**증상**
```
HTTP Error: HTTP GET error on 'https://d37ci6vzurychx.cloudfront.net/...' (HTTP 403 Forbidden)
X-Amz-Cf-Pop: ICN80-P4
X-Cache: Error from cloudfront
```

**원인**
EC2(ap-northeast-2, Seoul)의 IP 대역이 CloudFront 배포에서 차단됨.
`X-Amz-Cf-Pop: ICN80-P4` — 서울 POP 경유, EC2 IP 대역 특정 차단으로 추정.

**확인**
- 로컬 Mac(Seoul ISP) → CloudFront: **200 OK**
- EC2(Seoul) → CloudFront: **403**
- CloudShell(Seoul) → CloudFront: **200 OK** ← IP 대역 차이

**해결 방향**
CloudShell에서 다운로드 가능 → CloudShell을 proxy로 활용:
```
CloudFront → CloudShell(Seoul) → 사용자 S3(ap-northeast-2) → EC2
```

## 5. `read_parquet()` 내부 서브쿼리 불가

**증상**
```
Binder Error: Table function cannot contain subqueries
```

**원인**
URL 목록을 동적으로 생성하려고 서브쿼리를 `read_parquet()` 인자로 전달 시도:
```sql
FROM read_parquet(
    (SELECT list(...) FROM generate_series(...))
)
```
DuckDB table function은 서브쿼리를 인자로 받지 않음.

**해결**
60개 URL을 직접 배열로 나열.
